How does browser handles javacript ?

Javascript works in client side that means it runs inside browser.What is that one thing that allows javascript to work inside browser ? The answer is Javascript engine.

Little history about javascript engine :-

The first developers of javascript is netscape.They were also the developer of javascript engine as well.Now netscape is converted to mozilla and now the engine is recognised as spider monkey.Spider monkey engine is to render javascript in mozilla firefox.

Similarly, the javasript engine for chrome is V8 engine.For microsoft edge, the javascript engine is known as chakraThough these javascript engines have different names, they all render javascript and work in similar pattern.

How does these javascript engine work?

We use a script tag while writing a javascript code.With the help of this script tag, the javascript code is fetched.The javsacript code can be fetched primarily from 3 resources.These are network, cache and service worker.

Network --> Javascript code is stored in a server where the website is uploaded.When the user access a website, the javasript code is fetched from network.When the user visits a website for the first time , then network is used.

Cache --> When a user visits a website multiple times, the javascript code gets store in cache memory in user's computer.When the user visits the website again , javasript is fetched from cache.

What happens after javasript is fethed?

One the javasript code is fethed from either network or cache, it gets downloaded is form of bytes.

This binary data gets decoded with the help of byte stream decoderThe byte stream decoder decodes byte into tokens.The tokens are sent to parser.

What are the types of parsers?

There are basically 2 types of parsers in any type of javascript engine.

1) Pre-parser --> It parses all the unecessary code that is not needed immideately for execution.

2) Parser --> It parses all the useful code that is needed immideately for execution.This significantly improves the rendering time and this makes javascript super fast.
