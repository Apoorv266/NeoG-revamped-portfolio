How does the browser handle JavaScript?

Javascript works on the client side which means it runs inside the browser. What is that one thing that allows javascript to work inside the browser? The answer is theJavascript engine.

A little history about javascript engine:-

The first developer of javascript is Netscape. They were also the developer of the javascript engine as well. Now Netscape is converted to Mozilla and now the engine is recognized as a spider monkey . Spider monkey engine is to render javascript in Mozilla firefox.

Similarly, the javascript engine for chrome is a V8 engine . For Microsoft edge, the javascript engine is known as chakra .Though these javascript engines have different names, they all render javascript and work in a similar pattern.

Here is a list of javascript engines for different browsers.

  • Charka - internet explorer
  • Tamarin - adobe flash
  • Hermes - for mobile apps
  • JavaScriptCore - Safari

How do these javascript engines work?

We use a script tag while writing a javascript code. With the help of this script tag, the javascript code is fetched. The javascript code can be fetched primarily from 3 resources. These are network, cache, and service worker.

Network --> Javascript code is stored in a server where the website is uploaded. When the user accesses a website, the javascript code is fetched from the network. When the user visits a website for the first time, then a network is used.

Cache -->When a user visits a website multiple times, the javascript code gets stored in cache memory in the user's computer. When the user visits the website again, javascript is fetched from the cache.

What happens after JavaScript is fetched?

Once the javascript code is fetched from either network or cache, it gets downloaded in the form of bytes.

This binary data gets decoded with the help of a byte stream decoder. The byte stream decoder decodes bytes into tokens. The tokens are sent to the parser.

What are the types of parsers?

There are basically 2 types of parsers in any type of javascript engine.

1) Pre-parser --> It parses all the unnecessary code that is not needed immediately for execution.

2) Parser --> parses all the useful code that is needed immediately for execution. This significantly improves the rendering time and this makes javascript super fast.

What happens after parsing?

A node is created as the parsing happens based on the tokens that have been passed to the parser by the byte stream decoder. A node looks like a tree-like structure similar to a DOM tree. This tree is known as an abstract syntax tree or AST.

The interpreter walks through the AST and generates the byte code. This byte code is sent to the optimizing compilerwhich generates highly optimized machine code with is finally understood by our browsers.
