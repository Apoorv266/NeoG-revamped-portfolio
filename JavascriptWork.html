How does the browser handle JavaScript?

Javascript works on the client side which means it runs inside the browser. What is that one thing that allows javascript to work inside the browser? The answer is theJavascript engine.

A little history about javascript engine:-

The first developer of javascript is Netscape. They were also the developer of the javascript engine as well. Now Netscape is converted to Mozilla and now the engine is recognized as a spider monkey . Spider monkey engine is to render javascript in Mozilla firefox.

Similarly, the javascript engine for chrome is a V8 engine . For Microsoft edge, the javascript engine is known as chakra .Though these javascript engines have different names, they all render javascript and work in a similar pattern.

API stands for Application Programming Interface.

How does these javascript engines work?

We use a script tag while writing a javascript code.With the help of this script tag, the javascript code is fetched.The javsacript code can be fetched primarily from 3 resources.These are network, cache and service worker.

Network --> Javascript code is stored in a server where the website is uploaded.When the user access a website, the javasript code is fetched from network.When the user visits a website for the first time , then network is used.

Cache --> When a user visits a website multiple times, the javascript code gets store in cache memory in user's computer.When the user visits the website again , javasript is fetched from cache.

What happens after javasript is fethed?

One the javasript code is fethed from either network or cache, it gets downloaded is form of bytes.

This binary data gets decoded with the help of byte stream decoderThe byte stream decoder decodes byte into tokens.The tokens are sent to parser.

What are the types of parsers?

There are basically 2 types of parsers in any type of javascript engine.

1) Pre-parser --> It parses all the unecessary code that is not needed immideately for execution.

2) Parser --> It parses all the useful code that is needed immideately for execution.This significantly improves the rendering time and this makes javascript super fast.

What happens after parsing?

A node is created as the parsing happens based upon the tokens that has been passed to the parser by byte stream decoder.A node looks like a tree like structure similar to DOM tree.This tree is known as abstract syntax tree or AST.

The interpreter walks through the AST and generates byte code.This byte code is send to optimizing compailerwhich generates highly optimized machine code with is finally userstood by our browsers.
